- New "Channel Input" screen — bind an EQ preset to a specific app. Detection runs two layers: the standard `OPEN_AUDIO_EFFECT_CONTROL_SESSION` broadcast (Spotify / Poweramp / AIMP / foobar2000) plus a reflected `dumpsys audio` parse for apps that don't broadcast (YouTube / Netflix / Chrome / games). Same dual approach Wavelet and Poweramp Equalizer use under the hood. Optional `NotificationListenerService` binding (toggle in-screen) unlocks the dump parse on more OEMs — we never read your notifications, the bind is just the privilege gate.
- New "Audio Output" screen — bind an EQ preset to a specific output device (BT headphones, USB DAC, wired headphones, speaker). Plug in your DAC and the bound preset auto-applies; switch to BT and a different preset takes over. The auto-switch is itself a toggle (default on) so you can pin a preset and ignore route changes if you prefer.
- New "Audio Effects Pipeline" screen — single-place view of the processing chain (Audio Source → Channel Input → Environmental Reverb → MBC → ... → Audio Output). Drag the middle cards to reorder; tap any card to open its editor. Priority pills on the Channel Input and Audio Output cards show which routing scope is actually driving audio right now (Session-based / System-wide + auto-switch / manual).
- Environmental Reverb effect wired up. All 10 parameters (decay time, decay HF ratio, reflections level + delay, reverb level + delay, diffusion, density, room level, room HF level) at the exact Android `EnvironmentalReverb` API ranges. Attaches per-session in Session-based routing mode, so the reverb only colours apps you've explicitly bound a preset to.
- New "EQ314" Quick Settings tile — toggle DynamicsProcessing on / off without opening the app. Foreground service starts and stops correctly whether MainActivity is alive or not, and the in-app power FAB syncs back when you return (issue #9).
- New "Skip system sounds" toggle (default on) — bypasses the global DP on notification / ringtone / alarm / voice-call / navigation / assistant streams. Fixes the cracking / distortion some users heard on those sounds when the 127-band pre-EQ + limiter chain ran against short transient-heavy audio. Trade-off: while a notification is firing, music briefly goes un-EQ'd — far better than the music + distorted-notification combination. Flip the toggle off in Channel Input if you want every stream EQ'd regardless (issues #20, #21).
- Removed unused `androidx.media3` dependencies that were silently pulling `ACCESS_NETWORK_STATE` ("View network connections") into the merged manifest (issue #24). The app never used media3 — leftover from an earlier capture-based design that never shipped.
- Notification permission is no longer required for the app to function. The foreground-service notification is silently suppressed on devices where the user denied notifications; everything else keeps working (issue #19).
- Two MBC crash fixes: adding a new MBC band no longer crashes; dragging the cutoff dotted lines past adjacent bands no longer crashes (issue #17).
