- DP processing now persists across phone reboot, app close, and force-stop. New BootCompletedReceiver auto-restarts the EQ engine on boot if it was on before shutdown. The foreground service stays alive even after toggling DP off so the persistent notification (and its Turn On / Turn Off action) is always available without opening the app (issues #28, #29).
- Persistent notification is now a true on/off toggle. Title flips between "Equalizer314: Online" and "Equalizer314: Offline", and the action button reads "Turn Off" / "Turn On" accordingly. Expanded body now shows three lines — Mode (System / Session / Device) · Preset · Device — so you can see at a glance what's driving your audio without opening the app.
- New "Filtered" / "Show All" chips in Channel Input's Apps section. Filtered keeps the media-relevant heuristic (default behaviour); Show All lists every launchable app — useful for binding presets to games and other apps that don't declare media contracts. System services / wallpaper backends / ad-services packages are correctly filtered out (issue #30).
- MBC settings now persist with the preset across cold app open. Previously the MBC master switch read "on" after restart but the per-band thresholds / ratios / attack / release weren't being pushed to DP until you nudged a slider — the engine was running with default neutral compressor values. Per-band params are now reapplied on every DP-start path (Power FAB, QS tile, boot receiver). Also fixed the small audio glitch that used to happen on the first MbcActivity entry post-restart (issue #32).
- Preamp now persists with presets on every load path. Device-bound presets auto-applied by output routing now push their saved preamp value through to live DP (previously the preamp was written to prefs but the live engine kept the previous device's preamp value). Per-app session bindings also honour preamp now — bound presets get their preamp applied via the DP input-gain stage (previously ignored entirely). The save side was always correct; only the load paths were missing the apply (issue #13).
- Selecting "(none)" for a device in Audio Output now leaves whatever EQ is currently loaded alone, rather than wiping it. Tried the alternate "(none) disables DP" interpretation but reverted — "(none)" semantically means "no rule for this device", so the previously-loaded preset stays active when that device routes in. A future explicit "Disable EQ" option would be a separate dropdown entry (issue #13).
- Per-device and per-app binding edits now apply immediately on the live DP. Changing a preset binding in Audio Output (current-device dropdown, per-row dropdown, or Forget device) or Channel Input (per-app dropdown) used to write the new pref but only kicked in after a disconnect/reconnect or app restart. Now the new binding takes effect on the next audio buffer via broadcast → coordinator → live DP push (issue #13).
- New Mode · Preset · Device status card on the main EQ graph, sitting just below the band-info card and styled to match. Reads "System · <preset> · <device>" / "Session · <preset> · <device>" / "Device · <preset> · <device>" depending on what's currently driving the EQ. Notification's expanded body shows the same three lines stacked (issue #13).
- EQ shelf fidelity improvement. The parametric-to-DP converter previously sampled shelf filters at only 4-6 log-spaced points around the corner frequency, leading to a noticeably saggy / wrong-shape response in measurements (DynamicsProcessing's linear interpolation couldn't reproduce the analytical RBJ shelf curve from such sparse data). Shelf branches now use 8-10 sample points with denser octave-half spacing through the bend region. Measured response should now hug the analytical shelf much more closely, especially below 1 kHz where the previous undersampling was most visible (issue #26).
- Preset rows in every dropdown (Audio Output device bindings, Channel Input app bindings) and in the main-screen custom-presets list now show the preset's preamp value (e.g. "-7.2 dB") underneath the name. Sentinel rows like "(none)" stay clean — no fake "0.0 dB" subtitle.
- Per-app session preset attach now correctly reads `preamp` from the preset JSON when building the session DP (was ignored).
